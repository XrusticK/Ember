# CLAUDE.md

Памятка для Claude Code (и людей), работающих в этом репозитории.

## Что это

**Ember** — Flutter-приложение (тестовое задание). Одна карточка-мысль в день
(цитата / совет / мантра); отметил «прочитано» — растёт ежедневный **стрик** (🔥).
Premium открывает полный архив, все темы и заморозку стрика. Стрик — механика
удержания, карточки — дешёвый и красивый контент поверх неё.

Это демо **под запись экрана** — оно должно всегда запускаться, не светить ключей в
кадре и не падать по ходу сценария. Полностью офлайн.

## Платформы

Целевые — **iOS и Android**. Также оставлен web — только для быстрого прогона в
браузере и smoke-теста компиляции (`flutter build web`). Папка windows удалена.

## Обязательное поведение (из ТЗ)

1. **Онбординг** (2 экрана), завершается кнопкой «Продолжить».
2. **Пейвол** — два плана, Месяц / Год (год дешевле, выбран по умолчанию).
   «Продолжить» *эмулирует* покупку (без реального биллинга) и пишет флаг в хранилище.
   Открывается по запросу (баннер / замок архива) — подписку можно купить в любой момент.
3. **Главный экран** — карточка дня + лента-архив.
4. **Сохранение подписки** — флаг `is_premium` персистентен; купил один раз — премиум
   остаётся после перезапуска. Хранилище — `shared_preferences`.

## Стартовый роутинг (ядро логики — держим в одном месте)

Подписка больше не блокирует вход (её можно купить в любой момент), поэтому роутинг
прост:

```
onboarding_done == false   -> Онбординг
иначе                      -> Главный экран
```

Живёт в одном месте — `AppState.resolveInitialRoute()` (используется в `app.dart`).
Не разносить условия по экранам.

## Ключевые продуктовые правила

- **«Мысль дня»** выбирается детерминированно по дню в году. Пока не прочитана —
  показывается большой картой + кнопка «Прочитал». После прочтения сворачивается в
  компактное подтверждение и **уходит в архив**.
- **Архив = только прочитанные карточки** (`read_quotes`), новые сверху. Каждый день
  карточка новая; прочитанные копятся. Архив бесплатный — это личная история.
- **Разные темы — разные шрифты** (см. `core/mood_style.dart`): Спокойствие — Lora,
  Фокус — Manrope, Мотивация — Montserrat, Стоицизм — Playfair Display; у каждой темы
  свой акцентный цвет.
- **Огонёк** — анимированная иконка (`widgets/ember_flame.dart`), не эмодзи; гаснет при
  стрике 0. Рост стрика показывает праздничный всплеск (`widgets/streak_celebration.dart`).
- **Пуши** — ежедневное локальное напоминание (`data/notification_service.dart`),
  включается в настройках; на web — no-op.

## Ключи SharedPreferences (контракт)

| ключ              | тип          | смысл                                     |
|-------------------|--------------|-------------------------------------------|
| `onboarding_done` | bool         | онбординг пройден, не показывать снова     |
| `is_premium`      | bool         | «куплено» — ведёт сразу на главный экран   |
| `streak_count`    | int          | сколько дней подряд карточка прочитана      |
| `last_open_date`  | String (yyyy-MM-dd) | дата последней отметки «прочитано»  |
| `favorites`       | List<String> | id избранных карточек                      |
| `selected_theme`  | String       | настроение, выбранное в онбординге          |
| `reminder_enabled`| bool         | включено ли ежедневное напоминание          |
| `reminder_hour`   | int          | час напоминания (0–23), по умолчанию 20     |
| `read_quotes`     | List<String> | id прочитанных карточек (архив), в порядке чтения |

Ключи — константы в `lib/core/constants.dart`. Сырые строки нигде больше не пишем.

## Правило стрика

При «прочитано» сравниваем сегодня с `last_open_date` (см. `lib/state/streak.dart`):
- тот же день → без изменений (идемпотентно)
- вчера → `streak_count++`
- раньше / null → `streak_count = 1`

Затем сохраняем сегодняшнюю дату. Сравниваем **только дату** (`yyyy-MM-dd`), без
времени и таймзон. Логика вынесена в чистую функцию и покрыта юнит-тестами.

## Архитектура

- **State:** `provider` (`ChangeNotifier`). Единственный `AppState` держит premium /
  onboarding / streak / favorites и только он трогает `PrefsService`.
- **Data:** `PrefsService` оборачивает SharedPreferences; `QuotesRepository` грузит
  `assets/quotes.json` в модели `Quote`. UI не читает prefs и ассеты напрямую.
- **Слои:** `core` (theme, constants) → `data` (service, repo, models) →
  `state` (AppState, streak) → `features` (экраны) → `widgets` (переиспользуемый UI).

```
lib/
  main.dart              # bootstrap: prefs + NotificationService, AppState, runApp
  app.dart               # MaterialApp + резолвер стартового маршрута
  core/
    constants.dart       # ключи prefs, планы, темы
    theme.dart           # палитра ember, базовый шрифт (Manrope)
    mood_style.dart      # шрифт + акцент под каждую тему
  data/
    prefs_service.dart
    quotes_repository.dart
    notification_service.dart  # локальные пуши (ежедневное напоминание)
    models/quote.dart
  state/
    app_state.dart       # ChangeNotifier — единый источник правды
    streak.dart          # чистая логика стрика (тестируемая)
  features/
    onboarding/onboarding_screen.dart
    paywall/paywall_screen.dart
    home/home_screen.dart
    settings/settings_sheet.dart   # смена темы + напоминания
  widgets/
    quote_card.dart      # карточка с шрифтом под тему + замок премиума
    streak_badge.dart
    ember_flame.dart     # анимированный огонёк
    streak_celebration.dart  # всплеск при росте стрика
    premium_banner.dart
assets/
  quotes.json            # 100 реальных цитат по 4 темам — офлайн
test/
  widget_test.dart           # юнит-тесты логики стрика
  onboarding_flow_test.dart  # widget-тест онбординга
```

## Зависимости

`provider`, `shared_preferences`, `google_fonts` (шрифты под темы),
`flutter_local_notifications` + `timezone` + `flutter_timezone` (пуши).

## Опциональный AI-бонус

Кнопка «Сгенерируй мысль под моё настроение» → Gemini API (модель **Flash** —
быстрая/дешёвая), на вход — `selected_theme`. Ключ передаётся через `--dart-define`
(читается `String.fromEnvironment`), **не коммитится** и не попадает в кадр. Строго
опционально: офлайн-путь на JSON полностью закрывает ТЗ.

## Конвенции

- Держим маленьким и читаемым — оценивают ясность, а не количество фич.
- Бизнес-логика (стрик, роутинг, флаг покупки) — в `AppState`/сервисах, тестируется
  без виджетов.
- Никаких сырых строк-ключей и магических чисел — всё в `core/constants.dart`.
- Не добавляем пакеты сверх `provider`, `shared_preferences` (+ `http` только если
  делаем Gemini-бонус) без причины.
- Коммиты — на русском языке.
